Project Proposal: Interactive Smartphone Calling AI Agent (100% Free-Tier & Serverless)
This document outlines the architecture, user experience, and technical implementation plan for a highly immersive, browser-based voice conversation portal. The application will feel exactly like a physical smartphone call—complete with realistic dialers, call screens, glowing audio wave animations, and post-call receipt summaries—operating on a 100% free-tier, serverless architecture hosted on Vercel.

🌟 The Core Concept: "Phone Call in a Browser"
Instead of requiring paid physical phone numbers (like Twilio DIDs), this application creates a hyper-immersive, high-fidelity virtual calling terminal directly in the browser. Users interact with a simulated smartphone on the screen. They can choose an AI personality, dial a number, hit "Call", watch the UI transition to an active call screen with live wave visualizers, and hold a natural voice conversation with an AI agent. When they hang up, they instantly receive a full transcript and a generative summary of the call.

📱 The User Experience (UX) Journey
The interface will be divided into two main panels: a Sleek Desktop Admin Panel (glassmorphic layout) on the right, and an Interactive Premium Smartphone Viewport on the left.

Phase 1: The Dialer & Contacts (Idle State)
Smartphone Screen: Displays a modern lock screen that swipes open to reveal a gorgeous dialer pad (circle numeric buttons) and a contact book.
Featured Contacts (Agents): Pre-configured AI personalities (e.g., "Sarah - English Speaking Partner", "Preet - Business Coach", "Rohan - Mock Technical Interviewer").
Call CTA: Clicking a contact or entering a number and hitting the green dial button starts the call.
Phase 2: Active Call Screen ("Ringing..." ➔ "Connected")
Transition Animation: The screen shifts into a dark-blur calling screen. The caller’s avatar pulses.
Audio Handshake: The browser prompts for microphone permission.
Status States:
"Ringing..." (accompanied by a subtle simulated ringtone).
"Connected" (the UI reveals an active timer starting at 00:00).
Active Audio Waveform: A dynamic, glowing SVG or Canvas visualizer that animates:
Soft breathing waves when nobody is speaking.
High-frequency active waves matching the pitch and volume of the AI agent when speaking.
User waves matching the microphone input level when the user is speaking.
Action Controls: Fully functional interactive buttons:
Mute Microphone: Toggles browser mic mute (visual icon state change).
Keypad: Slides open a mini-dialer to press numbers during a call.
Speaker: Toggles audio output visual state.
Red Hang Up Button: Disconnects the call session cleanly.
Phase 3: Post-Call Receipt Screen (Call Summary)
Slide-up Transition: Disconnecting the call slides open a premium "Call Completed" receipt page.
Metrics Card: Displays Total Duration (e.g., 04:12 mins), Silence Ratio, and Sentiment Indicator.
Generative Summary: A bulleted AI-generated summary of what was discussed and any "Action Items".
Conversation Timeline: A complete transcript styled as chat bubbles (Left: AI, Right: User) so the user can review their spoken words and the AI's responses.
🛠️ The Technical Stack (100% Free-Tier & Serverless)
To keep this project completely free of monthly charges and hostable on a standard Vercel hobby plan, we leverage native browser hardware APIs combined with ultra-high-speed free-tier serverless cloud APIs.

1. Hearing (Speech-to-Text / STT): Native Web Speech API
Engine: webkitSpeechRecognition or SpeechRecognition (built-in Chrome, Safari, Edge).
Cost: $0.00 / Unlimited Minutes.
Integration: Transcribes user speech locally inside the browser. It triggers silence-detection when the user pauses, sending the compiled text payload to our Vercel API.
2. The Brain (AI LLM): Groq Llama-3-8B API
Engine: Groq Cloud API (running Llama-3-8B-Instruct).
Cost: $0.00 / Free Tier (Up to 14,400 calls per day—essentially infinite for testing!).
Speed: ~150ms latency (Groq is the fastest LLM inference engine in the world. The AI generates responses in the blink of an eye, maintaining natural conversation flow).
3. Speaking (Text-to-Speech / TTS): Native Web Speech API
Engine: window.speechSynthesis (local high-quality OS voices).
Cost: $0.00 / Unlimited Characters.
Integration: Plays synthesized audio directly through the user's speakers.
Natural Voices: We will target Microsoft Natural Voices (on Edge) and Google/Apple Natural Voices (on Chrome/Safari) which sound exceptionally lifelike without requiring paid APIs like ElevenLabs.
4. Database (Call Logs & Transcripts): Neon Serverless Postgres
Engine: Neon Serverless Postgres connected via Drizzle ORM.
Cost: $0.00 / Free Tier (Up to 0.5 GiB storage—holds millions of chat transcripts!).
Integration: Stores call history, summaries, and transcripts, allowing the Admin Panel to fetch past records.
🎨 Visual Design & Modern Aesthetics
We will craft this app using Vibrant Premium Aesthetics:

Theme: Deep dark mode (#0B0B0C background) with energetic electric-orange (#FF5500) and cyber-green (#10B981) accents.
Glassmorphism: Backdrop blur layouts with semi-transparent borders (rgba(255, 255, 255, 0.05)) to create a floating, modern SaaS workspace feel.
Device Frame: A beautifully detailed iOS/Android phone frame centered on the page with realistic screen notch, status bar (Time, Battery, Signal bars), and rounded corners.
Canvas Waves: A custom Web Audio API visualizer using the HTML <canvas> element to draw smooth, glowing bezier curves that respond actively to audio frequencies.
